# Development Workflow — Root File Policy

目的: リポジトリルートを最小限に保ち、ドキュメント・テスト・スクリプトの配置ルールを明確化します。

ルール概要:
- ルートに置くもの（最小化）
  - `README.md`, `pyproject.toml`, `requirements*.txt`, ビルド設定（`CMakeLists.txt`）やメタファイル（`.gitignore`等）
  - リポジトリ全体で参照される小さなランナー/ユーティリティ（`scripts/` 下に置くことを推奨）
- ドキュメント
  - 技術メモ・長文の実装ノートは `docs/` または `archive/docs/` に置く。ルートに置かれた詳細メモはアーカイブ先へのポインタに差し替える。
- テスト
  - 単体テストは `python/tests/`（またはプロジェクトの標準テストディレクトリ）に置く。ルートに `test_*.py` を残さない。
- スクリプト参照の更新
  - CI やスクリプトがルートのファイルを直接参照している場合、参照先を新しいパスに更新する（`python/tests/` や `docs/` への移行を反映）。
- アーカイブ手順
  - 詳細メモを `archive/docs/` に移動してから、ルート版を短い参照ポインタに差し替える（コピー→元削除→ポインタ作成）。
- CI/テスト実行
  - GUI 関連のヘッドレス検証は `scripts/run_pytest_with_pyqt_stub.py` を使用する（CI に組み込むことを推奨）。

運用メモ:
- 大きなファイル（ビルド成果物・バイナリ）は `build/`, `archive/bin/` 等に保管し、ルートに置かない。
- 例外が必要な場合は PR で理由を記載して承認を得る。

この文書はルート整理作業の一環として追加されました（作成日: 2026-01-19）。
