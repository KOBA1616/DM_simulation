# Card Instance Identity Policy

作成日: 2026-04-13

## 目的

「同じカードが盤面に出る/出ない」を実装上どの単位で判定するかを明確化する。

## 結論

- ルール上の同一性は `card_id`（カード種別）
- 実行時オブジェクトの同一性は `instance_id`（カード実体）

このため、**同じ `card_id` のカードが複数体バトルゾーンに存在することは仕様上可能**です。
制御は `instance_id` 単位で行われます。

## 実装根拠

1. `CardInstance` は `instance_id` と `card_id` を別々に保持する。
2. ゾーン操作（検索・削除）は `instance_id` で実行する。
3. シナリオ初期化では `instance_id_counter++` で全カード実体へ一意IDを割り振る。
4. 所有者追跡 `card_owner_map` も `instance_id` をキーにする。

## 意図

- 同名カード複数体、進化元、コピー効果などで「種別は同じだが実体は別」を安全に扱うため。
- 誤って `card_id` だけで管理すると、同名カードの移動/破壊/対象選択で取り違えが起きるため。

## 再発防止ルール

- 新規ゾーン処理は必ず `instance_id` ベースで実装する。
- `card_id` で対象を引く処理を追加する場合は、最終確定時に `instance_id` へ解決する。
